/** \~russian @page page_protoitems Прототипы предметов

Прототипы предметов хранятся в текстовом виде в файлах в директории Server/Proto/Items/.
Редактируются с помощью редактора объектов.

Файл items.lst содержит список файлов, содержащих данные о прототипах предметов.
В составе SDK это файлы:
@code
ammo.fopro
armor.fopro
container.fopro
door.fopro
drug.fopro
generic.fopro
grid.fopro
key.fopro
misc.fopro
wall.fopro
weapon.fopro
@endcode

Каждый файл создан для хранения своего типа объектов, форматы прототипов каждого из типов объектов имеет различия.<br /><br />
<strong>Общие поля</strong>:<br />
<table>
<tr><td>Pid</td><td>Номер прототипа (должен быть уникальным для всех типов объектов)</td></tr>
<tr><td>Type</td><td>Определяет тип объекта</td></tr>
<tr><td>PicMapName</td><td>Изображение на земле</td></tr>
<tr><td>PicInvName</td><td>Изображение в инвентаре</td></tr>
<tr><td>Flags</td><td>Флаги объекта</td></tr>
<tr><td>SoundId</td><td>Идентификатор звука</td></tr>
<tr><td>Material</td><td>Тип материала</td></tr>
</table>

\b Примечание по поводу <em>PicMapName</em> и <em>PicInvName</em>:<br />
В настоящий момент (ревизия 396) движок использует библиотеку DevIL ( http://openil.sourceforge.net/ ),<br />
которая позволяет использовать графические файлы следующих форматов:<br />
@code
.bmp, .cut, .dds, .doom, .hdr, .gif, .ico, .jpg, .lbm, .mdl, .pal, .pbm, .pcd, .pcx, .pgm, .pic, .png, .ppm, .psd, .psp, .raw, .sgi, .tga
@endcode
Исключения:
@code
.jp2, .exr, .mng, .tif, .lcms, .utx, .wdp
@endcode
( http://openil.sourceforge.net/about.php )<br />
Кроме того движок так же может использовать графику таких игр как:<br />
<ul>
<li>Fallout 1/2 (.frm)</li>
<li>Fallout Tactics (.spr, .til, .zar)</li>
<li>Arcanum (.art)</li>
<li>Boldur's Gate, Icewind Dale, Planescape Torment (.mos, .bam)</li>
</ul>
Так же можно использовать текстовый @ref page_format_fofrm "формат fofrm", данный вормат может к примеру объединить несколько различных графических файлов в анимацию или добавить изображению сдвиги, если исходный формат файла этого не позволяет или его редактирование является трудоемким.<br />
Имена файлов могут содержать дополнительные параметры (все параметры можно комбинировать):<br />
Формат SPR:
<ul>
<li>Для загрузки определенной анимации из SPR Файлов можно указать ее имя после знака $ в имени файла, например:
Mutant<b>$CrouchAttackClubSwing</b>.spr<br />
Без указания берется первая анимация</li>
<li>Влияние на цвета:<br />
Что бы сделать смещение по цвету укажите в параметрах имени файла ('$') необходимые значения.<br />
Формат: fileName$[index,r,g,b]animName.spr (смещений [...] может быть сколько угодно)<br />
где index - номер части тела (0 - разное, 1 - кожа, 2 - волосы, 3 - броня, -1 - все слои);<br />
r,g,b - смещения по цвету для каждой компоненты, могут быть отрицательными.<br />
Пример: someSprite$[2,50,-30,-20]anim1.spr - загрузить спрайт someSprite.spr, c анимацией anim1 и смещением по цвету волос на RGB (50,-30,-20).
</li>
</ul>
Формат ART:
<ul>
<li>Выбор палитры: номер палитры указывается после знака $<br />Пример: dfmbnsba<b>$1</b>.art</li>
<li>Включение прозрачности элементов палитры (alpha = max(R, G, B)): $t</li>
<li>Зеркальное отражение: $v - вертикальное отражение, $h - горизонтальное отражение</li>
<li>fx[-y] - указывает номера загружаемых кадров(-а) и их направление, пример:<br />
anim1$f3-0.art - значит будут загружено 4 кадра 3, 2, 1, 0</li>
</ul>
Формат BAM:
<ul>
<li>Для формата BAM есть возможность указания номера воспроизводимого цикла и кадра - для указания номера цикла в параметрах имени файла ('$') укажите его индекс, а для указания кадра добавьте прочерк и индекс кадра.
Примеры:<br />
fileName$5.bam - загрузить fileName.bam с кадрами под циклом номер 5;<br />
fileName$1-4.bam - загрузить fileName.bam с кадром номер 4 под циклом номер 1;<br />
fileName.bam - загрузить fileName.bam с кадрами под циклом номер 0.</li>
</ul>

\b Примечание:<br />
Если вы используете параметры в именах графических файлов, вам так же необходимо добавить их в файл HashNames.lst находящийся в папке Data клиента.<br />
Формат записи: [любое число] [имя файла] [конец строки].<br />
Например:
@code
0 art\scenery\BrazureFire$t.ART
0 art\scenery\BrazureFire$t3.ART
@endcode
