/**

\~russian @page commands Базовые команды серверу

@section commands_info Общая информация

Данные команды нужны для облегчения тестирования, отладки игрового процесса. Текущий набор является базовым, встроенным в движок, но с помощью скриптов вы можете существенно расширить функционал до необходимых вам пределов.

Для запуска команды перед ней должен стоять знак "~" (тильда). Так же возможен вызов команды по номеру, для этого наберите два данных знака, а затем номер команды.\n
После отправки команды на выполнение сервер ответит вам о результате выполнения операции.

@section commands_list Список команд

@par 1. exit\n
Уровень доступа: любой.\n
Выход из игры.

@par 2. myinfo\n
Уровень доступа: любой.\n
Запрос отображения некоторой информации о текущем персонаже - идентификатор, имя, пароль, уровень доступа.

@par 3. gameinfo \<type\>\n
Уровень доступа: модератор.\n
Запрос отображения информации об игре, где тип запрашиваемой информации (type) может быть следующим:\n
- 0) статистика использования памяти;\n
- 1) сводка по текущим игрокам в игре;\n
- 2) статистика локаций и карт;\n
- 3) текущие временные события;\n
- 4) текущее состояние данных @ref AnyData;\n
- 5) статистика предметов в игре;

@par 4. id \<name\>\n
Уровень доступа: модератор.\n
Запрос идентификатора криттера по его имени (name).
@remarks На данный момент не активна.

@par 5. movecrit \<id\> \<hx\> \<hy\>\n
Уровень доступа: модератор.\n
Передвинуть персонажа (id) в определенный гекс (hx, hy).

@par 6. kill \<id\>\n
Уровень доступа: модератор.\n
Убить персонажа (id).

@par 7. disconnect \<id\>\n
Уровень доступа: модератор.\n
Отсоединить персонажа (id) от сервера.

@par 8. toglobal\n
Уровень доступа: тестер.\n
Выйти текущим персонажем на глобальную карту.

@par 9. respawn \<id\>\n
Уровень доступа: модератор.\n
Воскресить персонажа (id). Если id равен 0, то значит воскресить текущего персонажа.

@par 10. param \<type\> \<num\> \<value\>\n
Уровень доступа: тестер.\n
Изменить параметр у текущего персонажа.\n
type должен быть равен нулю, num номеру параметра, value его новому значению.

@par 11. getaccess \<access name\> \<access password\>\n
Уровень доступа: любой.\n
Изменить уровень доступа.\n
access name - имя доступа (client, tester, moder, admin).\n
access password - пароль доступа.

@par 12. crash\n
Уровень доступа: имплементор.\n
Уронить серверное приложение.

@par 13. additem \<hx\> \<hy\> \<pid\> \<count\>\n
Уровень доступа: модератор.\n
Добавить предмет (pid, count) на определенный гекс (hx, hy).

@par 14. additemself, ais \<pid\> \<count\>\n
Уровень доступа: модератор.\n
Добавить предмет (pid, count) в инвентарь.

@par 15. addnpc \<hx\> \<hy\> \<dir\> \<pid\>\n
Уровень доступа: модератор.\n
Добавить НПЦ (pid, dir) на определенный гекс (hx, hy).

@par 16. addloc \<wx\> \<wy\> \<pid\>\n
Уровень доступа: модератор.\n
Добавить локацию (pid) в определенную точку (wx, wy) на @ref term_global_map "глобальной карте".

@par 17. reloadscripts\n
Уровень доступа: администратор.\n
Перезагрузить все серверные скрипты.

@par 18. loadscript, load \<script name\>\n
Уровень доступа: администратор.\n
Загрузить или перезагрузить определенный серверный скрипт (script name).

@par 19. reloadclientscripts, rcs\n
Уровень доступа: администратор.\n
Перезагрузить клиентские скрипты.

@par 20. runscript, run \<module name\> \<func name\> \<param0\> \<param1\> \<param2\>\n
Уровень доступа: администратор.\n
Запустить специальную серверную функцию (module name, func name), с определенными параметрами (param0, param1, param2).

@par 21. reloadlocations\n
Уровень доступа: администратор.\n
Перезагрузить прототипы локаций.

@par 22. loadlocation \<pid\>\n
Уровень доступа: администратор.\n
Загрузить определенный прототип (pid) локации.

@par 23. reloadmaps\n
Уровень доступа: администратор.\n
Перезагрузить прототипы карт.

@par 24. loadmap \<pid\>\n
Уровень доступа: администратор.\n
Загрузить определенный прототип (pid) карты.

@par 25. regenmap\n
Уровень доступа: администратор.\n
Перегенерировать карту на которой в текущий момент находится персонаж.
@remarks На данный момент работает со сбоями.

@par 26. reloaddialogs\n
Уровень доступа: администратор.\n
Перезагрузить все диалоги.

@par 27. loaddialog \<dialog name\> \<dialog id\>\n
Уровень доступа: администратор.\n
Загрузить определенный диалог (dialog name), под определенным идентификатором (dialog id).

@par 28. reloadtexts\n
Уровень доступа: администратор.\n
Перезагрузить все MSG файлы.

@par 29. reloadai\n
Уровень доступа: администратор.\n
Перезагрузить сумки НПЦ (Bags.cfg).

@par 30. checkvar, cvar \<tid var\> \<master is npc\> \<master id\> \<slave id\> \<full_info\>\n
Уровень доступа: модератор.\n
Проверить состояние определенной игровой переменной.\n
tid var - номер переменной;\n
master is npc - является ли master id от НПЦ, 0 - нет, остальное - да. Если да, то к master id автоматически добавляется 5kk, т.к. индексация НПЦ начинается с этого числа;\n
master id - первичный идентификатор;\n
slave id - вторичный идентификатор;\n
full_info - выводить ли полную информацию по переменной, 0 - нет, остальное - да.

@par 31. setvar, svar \<tid var\> \<master is npc\> \<master id\> \<slave id\> \<value\>\n
Уровень доступа: модератор.\n
Установить состояние определенной игровой переменной.\n
tid var - номер переменной;\n
master is npc - является ли master id от НПЦ, 0 - нет, остальное - да. Если да, то к master id автоматически добавляется 5kk, т.к. индексация НПЦ начинается с этого числа;\n
master id - первичный идентификатор;\n
slave id - вторичный идентификатор;\n
value - новое значение переменной.

@par 32. settime \<tmul\> \<year\> \<month\> \<day\> \<hour\> \<minute\> \<second\>\n
Уровень доступа: администратор.\n
Изменить игровое время.\n
tmul - ускорение времени, 0 - не менять;\n
year - год, 0 - не менять;\n
month - месяц, 0 - не менять;\n
day - день, 0 - не менять;\n
hour - час;\n
minute - минута;\n
second - секунда;\n

@par 33. ban \<add,add+,delete,list\> \<user\> \<hours\> \<comment\>\n
Уровень доступа: модератор.\n
Забанить определенного игрока.\n
add - добавить в бан персонажа;\n
add+ - добавить в бан персонажа и IP адрес;\n
delete - удалить из списка забаненных;\n
list - вывести список забаненных;\n
user - имя персонажа;\n
hours - время блокировки, в часах;\n
comment - комментарий, можно не писать.

@par 34. deleteself \<password\>\n
Уровень доступа: любой.\n
Удалить текущего персонажа.

@par 35. changepassword, changepass \<password\> \<new password\>\n
Уровень доступа: любой.\n
Изменить пароль для текущего персонажа.

@par 36. dropuid, drop\n
Уровень доступа: тестер.\n
Сбросить антиальт таймер для данного персонажа, что бы можно было перезайти за другого персонажа без ожидания истечения таймаута.

 */
