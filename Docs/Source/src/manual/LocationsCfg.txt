/** \~russian @page page_locationscfg Locations.cfg

В данном файле находится список всех локаций в игре.

Им в этом файле можно присваивать следующие параметры:

@b name \n
Имя локации. Не обязательный параметр.

@b size* \n
Размеры локации. \n
Рекомендуемые значения: 6 (small), 12 (medium), 24 (large). \n
По умолчанию - 24 (large).

@b max_players \n
Максимальное количество игроков, способных одновременно находиться на локации. \n
Целое беззнаковое значение, при нуле количество не ограничивается. \n
По умолчанию - 0.

@b map_X \n
Карты в локации. Вместо X подставляется порядковый номер карты. Отсчет с 0. X принимает значения от 0 до 49.\n
В конце строки прописывается идентификационный номер локации (смотрите Server\\scripts\\_maps.fos - блок Maps).\n
Символ '*' в конце имени карты означает возможность просмотра карты в меню Automaps.

@b entrance \n
Точки доступа в локацию.\n
Задается двумя способами:\n
1. Число, которое обозначает на сколько карт в списке (map_X) можно входить с глобальной карты. Гекс-вход равен 0.\n
2. '$' и далее, через запятую, перечисляются номер карты и номер гекса-входа на карту.\n
Например: "entrance=$0 0, 1 1, 0 3" будет означать, что существует три входа на локацию - на карту 0, в точку 0; на карту 1, в точку 1; на карту 0, в точку 3.\n
По умолчанию - 1.\n
Названия точек входа и их координаты на "карте" локации присваиваются в файле FOGM.MSG.

@b entrance_script \n
Скрипт для проверки точек доступа к локации.\n
По умолчанию - ничего.

@b visible* \n
Видна ли локция на глобале.\n
По умолчанию - 0.

@b auto_garbage* \n
Удалять карту, когда на ней не осталось ни одного игрока, местного НПЦ и предмета с флагом ГЕКК (актуально при geck_visible = 1).\n
По умолчанию - 1.

@b geck_visible* \n
Влияют ли на видимость локации расположенные в ней предметы с флагом ГЕКК, если да, то при количестве больше нуля локация получает свойство, аналогичное Visible.\n
По умолчанию - 0.



Максимальное количество локаций в игре - 500.



Пример городской локации:

@code
[Area 1]
name            = The Den
size            = 24
map_0           = den*          6
map_1           = den_carstop*  7
map_2           = repl_bank_den 210
entrance        = $0 0, 1 1, 0 3
visible         = 1
auto_garbage    = 0
@endcode

На этой локации есть три карты. Вход осуществляется на карты 0 (две точки входа) и 1 (одна точка входа).\n
Локацию видно на глобале. Она не удаляется, если там не осталось игроков.


Пример локации-энкаунтера:

@code
[Area 101]
name=Desert 1
size=6
map_0=e_desert1
geck_enabled=1
@endcode

На этой локации одна карта. Она будет удалена, когда ее покинет последний игрок.\n
Она невидима, если только там не находится предмет с флагом ГЕКК (например, машина).

 */
